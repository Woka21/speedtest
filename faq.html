FAQ – Internet Speed Test

30+ questions answered. Accuracy, privacy, Wi-Fi, upload, embed, troubleshooting and more.

Is the test accurate?

±5 %. We download 100 MB from Cloudflare edge, discard TCP slow-start and report the 90th-percentile speed – the rate you can actually sustain.

Do you store my IP or results?

No. Everything happens in your browser; no data hits our servers. See our privacy policy.

Which browsers are supported?

Any browser with fetch and ReadableStream (Chrome 63+, Firefox 57+, Safari 12+, Edge 79+).

Why is my speed lower than advertised?

ISP ads show “up to” speeds. Real factors: Wi-Fi signal, device limits, network congestion, background apps, server peering. Test on cable for the closest figure.

Wi-Fi vs cable – which should I trust?

Use cable (Ethernet) for the absolute figure; Wi-Fi shows real-life device performance.

What is a good internet speed?

25 Mbps down / 3 Mbps up is broadband. 100+ Mbps down is good for 4K streaming and large downloads.

How can I improve my Wi-Fi speed?
  1. Place router centrally and elevated
  2. Use 5 GHz or 6 GHz for less congestion
  3. Update firmware regularly
  4. Limit background apps during tests
Does the test measure upload speed?

Currently download only. Upload test is planned – we’ll POST the same 100 MB back to Cloudflare’s __up endpoint.

What is ping (latency)?

Ping = round-trip time (RTT) to server. ≤ 30 ms is excellent for gaming; 30-60 ms is good; > 100 ms may cause lag.

Can I embed the test on my site?

Yes! MIT-licensed. Link or iframe welcome. Repo: GitHub

Why do results vary during the day?

Network congestion, background updates, household device usage and ISP peering all fluctuate. Run 3 tests and average.

Is the test server near me?

Cloudflare has 300+ edges; your browser picks the closest automatically (usually < 50 km).

Does VPN affect the test?

Yes – VPN adds encryption overhead and longer routing. Disconnect VPN for the closest figure to your ISP plan.

How often should I test?

Once after setup, then weekly or whenever you feel slowdowns. Keep a log to show your ISP if needed.

Can I run the test on my phone?

Yes – use Wi-Fi for mobile broadband or disable mobile data to test home Wi-Fi. Works in any modern mobile browser.

What ports does the test use?

HTTPS 443 (HTTP/3 if available). No exotic ports – passes every firewall.

Does ad-blocker affect the test?

No – we don’t serve ads or tracking scripts; ad-blockers have nothing to block.

Why is my upload speed missing?

Upload test is under construction. For now use Cloudflare’s full suite or Fast.com.

Can I download my results?

Not yet. We’ll add a “Copy JSON” button soon – stay tuned on GitHub.

Does the test work offline?

No – you need an internet connection to reach Cloudflare edge.

Is the code open source?

Yes – MIT license. Fork, modify, embed freely. Repo link in footer.

How can I contact you?

Open an issue on GitHub or email via the address listed in the repo.

Does the test use cookies?

No cookies, no localStorage, no fingerprinting.

Can I run the test inside an iframe?

Yes – no X-Frame-Options header is set. Attribution appreciated.

What is the maximum speed the test can measure?

Theoretical limit is ~15 Gbps per stream (browser + Cloudflare). For higher, open multiple tabs or use the desktop app.

Why does the gauge reset after 200 Mbps?

Visual scaling. The circle reaches 100 % at 200 Mbps to keep the UI readable for the majority of users.

Does IPv6 work?

Yes – Cloudflare supports IPv6. Your device picks the best stack automatically.

Can I run the test on a smart TV?

If the TV browser supports fetch and readableStream, yes. Otherwise cast from phone.

Will you add jitter and packet-loss?

Yes – WebRTC stats are planned for a future release (no ETA yet).

How do I report a bug?

Open an issue on GitHub with browser version, OS and screenshot.
